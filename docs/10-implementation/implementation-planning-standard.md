# Cluely — Implementation Planning Standard

## Purpose

Every implementation begins with a short engineering plan.

Planning prevents unnecessary refactoring and reduces architectural drift.

---

# Before Writing Code

Answer:

* What problem is being solved?
* Which ADRs apply?
* Which design documents apply?
* Which files are expected to change?
* Which existing components will be reused?
* Which tests need to be added?
* What are the main risks?
* What alternatives were considered?

---

# Complexity Review

Estimate:

* Low
* Medium
* High

High-complexity work should be broken into smaller implementation phases whenever practical.

---

# Success Criteria

Define measurable completion criteria before implementation begins.

---

# Output

Every implementation should start with a brief implementation plan and finish by confirming that the planned scope was achieved without introducing architectural regressions.
